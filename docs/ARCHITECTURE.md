# 架构与项目结构

## 整体架构

前后端分离，通过 REST API 通信。

```
Frontend ──HTTP/JSON──▶  Backend  ──▶  Database（可能的）
```

## 后端结构

```
backend/
├── app/
│   ├── main.py                 # FastAPI 入口
│   ├── core/                   # 反馈算法、候选字生成、每日题目，含数据模型
│   ├── schemas/                # 请求/响应 Schema
│   ├── services/               # 业务和路由
│   └── db/                     # 数据库（可能的）
├── data/                       # 成语原始数据
└── tests/                      # 测试
```

## 数据模型

- `idioms`: 成语表（成语、拼音、释义、字频、是否常用）
- `daily_puzzles`: 每日题目表（日期、期号、关联成语）
- `games`: 游戏记录表（game_id, 玩家, 模式, 难度, 目标成语, 猜测记录, 状态）

## 核心算法

### 反馈算法
1. 先标记位置正确的字为绿色，消耗目标字
2. 剩余字中，若目标成语还有该字则标黄，否则标灰

### 候选字生成
1. 取目标成语 4 字
2. 按难度策略生成干扰字（随机/高频/迷惑）
3. 合并打乱返回
